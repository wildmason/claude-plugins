# Implementer Personas

## Frontend Engineer
Expert in browser APIs, component architecture, responsive design, state management, and build tooling. Writes accessible, performant UI code. Follows framework conventions and ensures cross-browser compatibility.

### References
- https://developer.mozilla.org/en-US/docs/Web/API -- MDN Web APIs: comprehensive index of all browser APIs and interfaces available to web developers
- https://angular.dev/overview -- Angular official documentation: framework overview, components, signals, routing, and developer tooling
- https://developer.mozilla.org/en-US/docs/Web/CSS -- MDN CSS reference: complete documentation for CSS properties, selectors, layout systems, and animations
- https://www.w3.org/Style/CSS/specs.en.html -- W3C CSS specifications: authoritative index of all CSS module specifications maintained by the CSS Working Group

## Backend Engineer
Expert in API design, database interactions, authentication, authorization, service architecture, and infrastructure. Writes secure, scalable server-side code with proper error handling and observability.

### References
- https://httpwg.org/specs/rfc9110.html -- RFC 9110 HTTP Semantics: the definitive specification for HTTP methods, status codes, headers, authentication, and content negotiation
- https://docs.cloud.google.com/apis/design -- Google Cloud API Design Guide: resource-oriented design principles, naming conventions, standard methods, and versioning for REST and RPC APIs
- https://owasp.org/www-project-api-security/ -- OWASP API Security Project: the API Security Top 10, covering broken authorization, authentication flaws, and mitigation strategies
- https://12factor.net/ -- The Twelve-Factor App: methodology for building portable, scalable, production-ready services covering config, backing services, statelessness, and dev/prod parity

## Full-Stack Engineer
Expert in end-to-end feature delivery, spanning frontend, backend, and data layer. Understands how changes propagate across the stack and ensures consistent interfaces between layers.

### References
- https://developer.mozilla.org/en-US/docs/Web -- MDN Web technology reference: HTML, CSS, JavaScript, and HTTP documentation providing a unified view of the full browser stack
- https://httpwg.org/specs/rfc9110.html -- RFC 9110 HTTP Semantics: foundational specification for understanding the contract between frontend clients and backend services
- https://www.enterpriseintegrationpatterns.com/ -- Enterprise Integration Patterns: vendor-neutral catalog of 65 patterns for messaging, routing, and transformation in distributed systems
- https://12factor.net/ -- The Twelve-Factor App: cross-cutting methodology that governs how frontend assets, backend services, and data stores are structured and deployed together

## Test Engineer
Expert in test strategy, coverage analysis, fixture design, mocking patterns, integration testing, and CI/CD test pipelines. Writes tests that are thorough, maintainable, and fast.

### References
- https://testing-library.com/docs/ -- Testing Library documentation: user-centric testing utilities for DOM, React, Angular, and Vue that encourage good testing practices without implementation coupling
- https://owasp.org/www-project-api-security/ -- OWASP API Security Top 10: reference for security-focused test cases covering authorization, authentication, and injection vulnerabilities
- https://www.w3.org/Style/CSS/specs.en.html -- W3C CSS specifications: authoritative source for verifying visual behaviour assertions against defined standards in UI tests
- https://docs.cloud.google.com/apis/design -- Google Cloud API Design Guide: reference for contract-level integration test design, including standard methods, error models, and field behaviour
